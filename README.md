VPT Learn
Образовательная платформа для изучения языков программирования и выполнения практических заданий.
Описание проекта
VPT Learn — мобильная и веб-платформа, предназначенная для хранения данных о языках программирования и практических задачах, связанных с ними. Реализована современная архитектура с акцентом на модульность, тестируемость и масштабируемость. В проекте реализованы как клиентская часть (Flutter), так и серверная часть (ASP.NET Core 8.0) с использованием Supabase в роли БД и механизма аутентификации.
Цели и задачи
разработать мобильное приложение и backend, способные хранить данные о языках программирования и практических заданиях;
провести анализ существующих решений и определить требования к системе;
разработать техническое задание и прототип интерфейса;
спроектировать архитектуру базы данных и backend-логику;
реализовать frontend и backend;
написать модульные и интеграционные тесты;
протестировать систему на целевой группе пользователей и собрать обратную связь.
Архитектура проекта
Общая архитектура: N-Layer монолит с разделением на слои и домены (курсы, уроки, задания, прогресс, уведомления, пользователи). По мере роста возможно эволюционное выделение микросервисов.
Клиентская часть: Flutter (Dart), модульная архитектура, декларативный UI на Material Design 3.
Серверная часть: ASP.NET Core 8.0, RESTful API, слои Presentation (HTTP-контроллеры), Domain (бизнес-логика), Data (модели БД), вспомогательные утилиты, DI, логирование.
База данных и аутентификация: Supabase (PostgreSQL) с использованием Supabase Auth и RLS. JWT-токены с поддержкой refresh-токенов.
Интеграции: Supabase PostgREST, GoRouter на клиенте, реальное время через подписки Supabase.
Технологии
Клиент: Flutter, Dart
Сервер: ASP.NET Core 8.0, C#
База данных и аутентификация: Supabase (PostgreSQL, Supabase Auth, RLS)
Архитектура: N-Layer монолит (поправки к слоистой архитектуре)
Тестирование: модульное, интеграционное и пользовательское тестирование
Контейнеризация и развёртывание: Docker (песочницы для C/C#/Java/Python), VPS Ubuntu 24.04
Основные функциональные требования
Регистрация и вход пользователей (JWT, refresh-токены)
Аутентификация и авторизация (права: student, admin)
Просмотр каталога курсов и их описаний
Навигация по курсам, модули и уроки с теоретическим материалом и заданиями
Управление пользователями (для администраторов)
Безопасность: шифрование паролей, валидация на клиенте и сервере, обработка ошибок
Производительность: кэширование статических ресурсов, оптимизация запросов к БД
Масштабируемость: модульная архитектура, REST API, возможность добавления новых модулей
Архитектура базы данных и API
ER-диаграмма отражает сущности: users, courses, lessons, exercises, submissions, progress, notifications
Серверная часть структурирована в слои:
Presentation: HTTP-контроллеры (DTO, валидация)
Domain: бизнес-логика (проверки, прогресс, доступ)
Data: доступ к данным через Supabase модели
Utils/DI/Logging: вспомогательные утилиты и внедрение зависимостей
API endpoints (пример):
Auth: register, login, signout
User: current, make-admin
Course: allcourses
Lessons: allcourses/{courseId}/lessons, alltasks/{lessonId}
Admin: users/update-user-auth
Безопасность: JWT Bearer токены, Supabase Auth, RLS, Swagger UI с Bearer auth
Реализация и сборка
Клиентская часть: Flutter SPA, Material 3, GoRouter, ListView.builder, жизненный цикл через события
Серверная часть: ASP.NET Core 8.0, Swagger UI, контейнеризация под песочницы
Интеграция: клиентская часть взаимодействует с Supabase Auth для логина/регистрации, далее через JWT Bearer к API, который проксирует запросы в Supabase (PostgREST + RLS)
Тестирование
Модульное тестирование (Unit): тесты контроллеров и сервисов с моками Supabase
Интеграционные тесты (API): проверка middleware, RLS, контейнеров песочниц
Пользовательские тесты (E2E): тесты на эмуляторах Android/iOS (GoRouter, состояния loading/error)
Результаты: покрытие > 90% по M1-M5, I1-I5, U1-U5
Развертывание и эксплуатация
Сервер: VPS Ubuntu 24.04 (пример: 89.23.97.76:5000)
Контейнеризация: Docker-песочницы для разных языков (C/C#/Java/Python)
База данных и аутентификация: Supabase (PostgreSQL) с JWT proxy
Клиент: Flutter приложение, работающее через Supabase Auth и REST API
Запуск сервера: dotnet run --Release
